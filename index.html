A importância do HTML Semântico em nossas vidas

Quando utilizo o html semântico, eu consigo olhar no meu código e saber o que cada coisa é, se eu fosse colocar uma div e depois fazer um nesting de divs um dentro da outra, o código ficaria uma bagunça depois. Porém se eu utilziar uma main, colocar asides, colocar sections e outras semânticas, eu consigo saber o que cada um faz de maneira simplificada

Motivos por trás de sua criação

O html semântico surgiu a partir do HTML5, as tags semânticas justamente foram feitas com o propósito de retirar a redudância de utilizar uma ou duas tags para quase tudo, além de permitir uma formatação padronizada em qualquer aplicação, e até mesmo no css, que por sinal já é uma bagunça por si só

Como utilizo Classes

Dependendo da aplicação, eu começo com um nav e um header, a nav para navegação e a logotipo e o header para conteúdo embaixo, a partir disso passamos para o main e seus asides, onde o conteudo principal fica centralizado e os asides são utilizados para anúncios, informações adicionais ou conteúdo extra. O footer serve como um rodapé para colocar as informações sobre meu e-mail, contatos entre outros

Gostei de usar HTML semântico?

Sim, ele é prático, intuitivo e não se acumula como uma bola de neve, igual encher de divs tudo.

Como aprendi sobre html semântico
